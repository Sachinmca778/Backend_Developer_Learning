My Interview Experience at InfoEdge (Naukri.com)
Role: Backend Developer
Location: Onsite Hiring Drive | Jan 16

This was one of the most spontaneous interviews I gave.
I received a Google Form from a recruiter after uploading my resume on Naukri. Within a week, I was shortlisted.

Round 1: DSA (45 min)
Questions asked:
 → Three Sum Problem
 → LCA (Lowest Common Ancestor) of Deepest Leaves
I explained the brute force approach, then optimized using sorting and two pointers. 
All code was written on paper.
For LCA, I first explained a two-step (BFS + DFS) approach, then optimized to a single DFS after a hint from the interviewer.

Feedback received from the interviewer:
 → Strong problem-solving and optimization skills
 → Good communication of the approach
 → Need to be more careful with Java syntax (used to solving in JavaScript)

Round 2: System Design (45 min)
I was given a choice between explaining:
→ My current project architecture
→ A system like a URL shortener

I started with my project, but it was too complex for the format. Switched to designing a URL shortener.

Topics covered:
 → Requirement breakdown
 → API design
 → DB selection
 → Hashing & collision handling
 → Code structure (POJO, Controller, Service, Repository)
 → Singleton pattern
 → Thread safety and synchronization

Feedback:
 → Clear understanding of code-level design
 → Strong multithreading fundamentals
 → Need to improve clarity in high-level system design

Round 3 (Final Round): DSA + Managerial
DSA Question: Evaluate Expression (stack-based)
 → Explained the approach and walked through test cases

Then we moved on to:
 → Why I want to join
 → Career aspirations
 → Developer mindset
 → Evolution of tech

Feedback:
 → All rounds marked positive
 → Strong DSA and multithreading skills
 → System design needs more depth
 → HR said they’d reach out when new roles open up
