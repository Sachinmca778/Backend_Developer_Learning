## Why Behavioral Round Matters

| | Tech Round | Behavioral Round |
|--|-----------|------------------|
| Filters for | Knowledge / problem-solving | Cultural fit, communication, growth potential |
| Decides | "Can do the job?" | "Should we hire / level / pay X?" |
| At senior levels | Both equally important | Often **deciding factor** |

> "Same tech score? **Behavioral wins**." — Common in companies with structured interview loops (Amazon LP, Google Googleyness, etc.).

## STAR Quick Reference

```
S — Situation:  Context (1-2 sentences)
T — Task:       Your responsibility / problem
A — Action:     What YOU did (most detail — 50% time)
R — Result:     Impact (quantified ideally)
```

**Time:** Aim for **2-3 minute** answers. < 1 min = thin. > 4 min = rambling.

## 1.8 yr PHP → Java Transition Framing

**Common interviewer concerns:**
- "Why switching from PHP to Java?"
- "Java production exp kitna hai?"
- "Spring Boot real project mein use kiya?"

**Strong framing:**
- **Self-driven learning** — show structured roadmap (this repo!)
- **Concrete projects** — REST API + JPA + Security side project
- **Bridging PHP exp** — "OOP, MVC, REST, DB modeling — concepts same; Java/Spring Boot platform shift"
- **Growth narrative** — "want to work on JVM-based scale + ecosystem (Kafka, microservices)"
